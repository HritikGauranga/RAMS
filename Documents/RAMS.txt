1. Product Overview
The Remote Alarm Monitoring System is an industrial monitoring and notification device designed for remote supervision of equipment, processes, and infrastructure. The system continuously monitors digital and analog field signals and automatically generates SMS notifications when configured alarm conditions occur.
Configuration and monitoring are performed through an embedded Web User Interface accessible through either Access Point (AP) Mode or Ethernet connectivity. The system also supports remote interaction through SMS commands for status reporting, alarm acknowledgement, output control, and configuration management.
The Remote Alarm Monitoring System has been designed as a flexible and cost-effective solution for applications requiring alarm notification, equipment supervision, and remote access without the complexity of a traditional SCADA system.


2. System Features
The system provides the following functionality:
Inputs
• 4 Digital Inputs for monitoring dry contacts, switches, and alarm signals
• 2 Analog Inputs (4–20mA) for process variable monitoring such as level, pressure, or flow
Outputs
• 2 Relay Outputs for remote control and alarm indication applications
Communications
• SMS Notifications for alarm, return, acknowledgement, and status reporting
• SMS Remote Configuration for modifying settings and controlling outputs remotely
• Ethernet Access for local network connectivity and web-based configuration
• Access Point Configuration Mode for direct wireless access during commissioning and maintenance
Storage
• Configuration Storage for retaining user settings and system parameters
• Event Logging using LittleFS for recording alarms, returns, acknowledgements, and system events ----> (Park for now)
Security
• Web Login Authentication to restrict access to authorized personnel
• Authorized Number List for trusted phone numbers with elevated SMS privileges
• PIN Protected SMS Commands for secure remote configuration and control
Monitoring
• Alarm Detection for digital and analog input alarm conditions
• Alarm Acknowledgement to confirm receipt and review of active alarms
• Alarm Return Detection to identify restoration of normal operating conditions
• Event Logging for maintaining a historical record of system activity and alarms

2.	The device enables its internal Wi-Fi Access Point.
3.	Connect a mobile phone, tablet, or computer to the device Wi-Fi network.
4.	Open a web browser.
5.	Navigate to the default address:
http://20.20.20.20
6.	Enter the administrator credentials to access the Web Interface.
Default Credentials:
Username: Admin & Password: Admin@123
Note:  The currently assigned IP address shall be displayed on the Dashboard page as “TCP END POINT”, You can use this IP to access WEB UI on you network. (When Ethernet is Connected)

6.5 Time to Alarm (TTA)
Defines the delay between detection of an alarm condition and activation of the alarm.
Purpose:
Prevents false alarms caused by:
• Contact Bounce
• Electrical Noise
• Short Duration Events
Range:
1 to 65535 Seconds

Example:
Input:
Generator Fault

Time to Alarm:
30 Seconds

Operation:
1.	Fault occurs
2.	Timer starts
3.	Condition remains active for 30 seconds
4.	Alarm becomes active
5.	SMS notification sent
If the fault clears before 30 seconds:
• Alarm is cancelled 
• No SMS is sent

6.6 Time to Return (TTR)
Defines the delay required before an active alarm returns to normal.
Purpose:
Prevents alarm chattering and repeated notifications.
Range:
1 to 65535 Seconds
Example:
Input:
Tank Float

Time To Return:
60 Seconds

Operation:
1.	Alarm active
2.	Float returns to normal
3.	Return timer starts
4.	Condition remains normal for 60 seconds
5.	Alarm cleared
6.	Return SMS transmitted

7.6 Alarm Types
Each Analog Input supports four alarm modes.
High Alarm
Alarm activates when the measured value rises above the configured Set Point.
Alarm clears when the value falls below the configured Reset Point.
Example:
Set Point = 80
Reset Point = 75
Operation:
Value ≥ 80
Alarm Active
Value ≤ 75
Alarm Cleared

Low Alarm
Alarm activates when the measured value falls below the configured Set Point.
Alarm clears when the value rises above the configured Reset Point.

Example:
Set Point = 20
Reset Point = 25
Operation:
Value ≤ 20
Alarm Active
Value ≥ 25
Alarm Cleared

In-Band Alarm
Alarm activates when the measured value enters a defined operating range.
Example:
Set Point = 40
Reset Point = 60
Alarm Active:
40 to 60
Alarm Cleared:
Outside configured range
Out-of-Band Alarm
Alarm activates when the measured value leaves a defined operating range.
Example:
Lower Limit = 40
Upper Limit = 60
Alarm Active:
Below 40 or Above 60
Alarm Cleared:
40 to 60
7.7 Set Point
Defines the value at which an alarm condition becomes active.
The Set Point interpretation depends on the selected alarm type.
Examples:
High Level Alarm:
Set Point = 4500 Liters
High Pressure Alarm:
Set Point = 8 Bar
Low Level Alarm:
Set Point = 10%
7.8 Reset Point
Defines the value required to clear an active alarm.
The Reset Point provides hysteresis and prevents repeated alarm activation caused by small fluctuations around the alarm threshold.
Example:
Set Point = 80%
Reset Point = 75%
Operation:
Value reaches 80%
Alarm Active
Value fluctuates between 79% and 78%
Alarm remains active
Value falls to 75%
Alarm Cleared

9. Alarm Management
The Alarm Management system is responsible for monitoring all configured Digital Inputs and Analog Inputs, generating alarm events, notifying users, controlling outputs, and maintaining alarm history. Each configured alarm operates independently and maintains its own status, timing parameters, acknowledgement state, and event history.
The Alarm Management system is designed to prevent nuisance alarms while ensuring critical events are reported reliably.
9.1 Alarm Identification
Every alarm generated by the system shall be assigned a unique Alarm ID.
Alarm IDs are used for:
• SMS Notifications
• Alarm Acknowledgement
• Event Logs
• Alarm History
Example:
Alarm ID: 101
Alarm Source: AI1
Alarm Type: High Level
9.2 Alarm States
An alarm may exist in one of the following states:
Normal
Input operating within configured limits.
No active alarm exists.
Pending Alarm
Alarm condition detected.
Time To Alarm timer running.
Alarm not yet active.
No notification transmitted.
Alarm Active
Alarm condition confirmed.
Alarm SMS transmitted.
Alarm event logged.
Optional relay actions executed.
Acknowledged
Alarm has been acknowledged by an operator.
Alarm condition still exists.
Alarm remains active until return condition occurs.
Pending Return
Return condition detected.
Time To Return timer running.
Alarm remains active.
No return notification transmitted.
Returned To Normal
Alarm condition cleared.
Return SMS transmitted.
Return event logged.
Alarm cycle completed.

11. SMS Command Structure
The Remote Alarm Monitoring System supports SMS-based interaction for status monitoring, alarm acknowledgement, relay control, and remote configuration. SMS commands are processed automatically upon receipt and validated according to the sender’s authorization level.
The SMS Command System provides four access levels:
• Public Access
• PIN-Protected Access
• Authorized Number Access
• Event Recipient Access
11.1 SMS Command Processing
Upon receiving an SMS message:
1.	SMS received
2.	Sender number identified
3.	Command parsed
4.	Authorization verified
5.	Command executed
6.	Response generated
7.	Event logged
Invalid commands shall generate an error response.
11.2 Access Levels
Public Access
Available to any phone number.
Permitted Operations:
• Status Requests
• Input Status Requests
• Output Status Requests
• Signal Status Requests
• Firmware Version Requests
Public users cannot:
• Modify configuration
• Control outputs
• Acknowledge alarms
PIN-Protected Access
Available to any phone number possessing the correct PIN.
Permitted Operations:
• Configuration Changes
• Output Control
• Alarm Acknowledgement
• Network Configuration
• Phone Number Management
Authorized Number Access
Phone numbers contained within the Authorized Number List.
Authorized Numbers may perform all operations without requiring a PIN.
Event Recipients
Event Recipients receive notifications but do not automatically receive control privileges.
